トラブルシューティング

よくある問題とその解決方法をまとめています。困ったときは、まず該当する項目を確認してください。

よくある問題と解決策

投票しようとすると「不正なリクエストです（nonce）」と出る

原因: アンケートを表示しているページがキャッシュされ、古いセキュリティトークン（nonce）が使われていることが多いです。

1

キャッシュプラグイン（WP-Optimize、WP Super Cache など）でキャッシュを削除する。

2

サーバー側やホスティング会社のキャッシュ、CDN（Cloudflare など）のキャッシュをパージする。

おすすめの対策は、アンケートを設置したページの URL をキャッシュ対象から除外することです。設定方法はご利用のキャッシュプラグイン／CDN の各ドキュメントを参照してください。

リバースプロキシ／CDN 配下で最初の1人しか投票できない

原因: CDN やリバースプロキシ経由だと全アクセスがプロキシの IP に見え、全員が同一 IP と判定されるためです。

1

wp-config.php に define( 'KASHIWAZAKI_POLL_TRUST_PROXY', true ); を追加し、実クライアント IP ヘッダーを信頼する設定を有効にする。

この設定は、信頼できるリバースプロキシ／CDN を経由している環境でのみ有効にしてください。経由していない環境で有効にすると、ヘッダー偽装で重複投票防止を回避される恐れがあります。

投票できない（重複投票と判定される）

原因: 同じ IP アドレスまたは Cookie で、すでに投票済みと判定されています。

1

ブラウザの Cookie をクリアして、再度投票を試す。

2

社内 LAN など共有 IP 環境では、他の人がすでに投票している可能性があります。全員に再投票してもらう場合は、基本設定の「投票制限の解除」を使います。

グラフが表示されない

原因: グラフ描画に使う Chart.js（CDN 経由）が読み込めていない可能性があります。

1

ブラウザの拡張機能（広告ブロッカー等）や、CDN をブロックする設定を一時的に無効にして確認する。

2

他の JavaScript のエラーで描画が止まっていないか、ブラウザのコンソールを確認する。

データセットページが 404 になる

原因: /datasets/ の URL ルール（リライトルール）が更新されていない可能性があります。

1

「設定」→「パーマリンク」を開き、何も変更せずに「変更を保存」を押す。これで URL ルールが再構築されます。

データセットファイル（CSV 等）が生成されない

原因: 投票データがない、またはファイル書き込み権限の問題が考えられます。

1

そのアンケートに1件以上の投票があることを確認する。

2

基本設定ページの「データセット一括生成」を実行して、まとめて生成し直す。

データファイルはアップロードディレクトリ配下に保存されるため、プラグインを更新してもデータが消えることはありません。

項目 要件
WordPress5.0 以上
PHP7.4 以上
グラフ描画Chart.js 4.4.8 / chartjs-plugin-datalabels 2.2.0（CDN 経由）

技術仕様

  • 投票データはカスタム投稿タイプ「データセット（poll）」の投稿メタとして保存
  • 重複投票防止: IP アドレス + Cookie の組み合わせ
  • データ形式: CSV / XML / YAML / JSON / SVG の5種類を自動生成（アップロードディレクトリ配下に保存）
  • 構造化データ: Schema.org Dataset 型 JSON-LD / Dublin Core / BreadcrumbList / DataCatalog
  • 専用サイトマップ: sitemap-poll-datasets.xml を自動生成
  • 外部通信: グラフ描画用の Chart.js を CDN から読み込み（サーバー間の外部送信はなし）
  • メール送信・定期実行（Cron）は使用しません
